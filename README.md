# 品牌全案策略指南 (Brand Strategy Guide)

> 基于《品牌全案策略指南 2.0》知识体系的 Claude Code Skill。通过深度对话式诊断，引导你完成从市场调研、战略定位到战略执行的完整品牌全案策略方案。每个关键策略判断必有真实商业案例佐证，最终输出为可直接交付的 HTML 网页 PPT 或 PPTX 文件。

---

## 谁适合用这个工具？

### 初创品牌创始人

你有一个产品想法，但不确定"这个品牌到底应该是什么"——目标用户是谁、跟竞品比差异在哪、定价怎么定、怎么让人记住你。这个 skill 不会给你"老板加油"式的鸡汤文案，而是通过系统性的追问和方法论推导，帮你从混沌中找到那条**有依据的策略线索**。22 轮深度对话（或直接填写 brief 问卷）之后，你会拿到一份 80-100+ 页的完整品牌全案，拿到就能用、能拿去见投资人。

### 想要转型品牌策划的设计师

你已经有视觉执行能力，但想往上游走——从"把别人的策略做成好看的图"变成"自己出策略"。这个 skill 内置的完整方法论体系（生态位分析、四种定位方法决策树、定价五层塔、四轮驱动传播模型等）可以成为你的**策略思维训练场**。每次使用都是一次跟着框架走一遍完整策略推导的机会，方法论和理论库会告诉你每一步背后的"为什么"。

### 准备创业做品牌的人

你看到了一个市场机会，但需要系统性地验证它是否成立——行业是否值得进入、竞争格局有没有缺口、用户有没有未被满足的真实需求、你的团队能不能吃得下。这个 skill 的市场调研模块会带你完成这四个步骤的交叉验证，**找到你的生态位**——那个市场空白、用户需求、自身能力三者交汇的位置。如果生态位不存在，skill 会如实告诉你，而不是编一个给你。

## 方法论体系

### 核心底层逻辑：生态位驱动的价值三段论

整个方法论体系建立在一个基本命题之上：**品牌策略的核心问题不是"怎么卖更多"，而是"我在生态中占据什么位置"。**

```
发现价值（市场调研）  →  创造价值（战略定位）  →  传播价值（战略执行）
      ↓                                              ↓
  找到生态位                                     商业化交付
      ↓                                              ↓
      └─────────── 循环反馈，持续校准 ←─────────────┘
```

三个阶段不是线性步骤，而是一个**价值正向循环**：

- **市场调研**的最终目的不是堆积数据，而是帮品牌找到**生态位**——市场中一个未被充分满足、且自身有能力占据的位置
- **战略定位**是基于生态位来构建品牌的身份——"我是谁"必须回答"我在生态中占据什么位置"
- **战略执行**是将生态位转化为可感知的用户价值，通过产品和传播让生态位从分析结论变成用户心智中的事实

生态位 = 市场空白 ∩ 用户需求 ∩ 自身能力。生态位找到之后，品牌才有清晰的"不做什么"的边界。

### 两个核心决策树

一切策略判断由两个不可跳过的决策树驱动：

**决策树一：市场调研 → 找到生态位（4 步漏斗）**

```
第1步：行业是否值得进入？
  ├─ 大市场+高增长+政策利好 → 继续
  ├─ 中等市场 → 谨慎进入（需强理由）
  └─ 小市场+萎缩 → 换赛道或等结构性窗口

第2步：竞争格局有没有缺口？
  ├─ CR5<30%（分散市场）→ 品类占位机会
  ├─ CR5 30-60%（集中化中）→ 找细分切口
  └─ CR5>60%（寡头市场）→ 只能走对立或边缘颠覆

第3步：用户有没有未被满足的需求？
  ├─ 多画像共有痛点+竞品无人解决 → 强生态位信号（P0）
  ├─ 部分画像有痛点+竞品已在解决 → 弱生态位信号（P1）
  └─ 无明显未满足需求 → 需差异化定位

第4步：我能不能吃得下？
  ├─ 能力完全覆盖 → 生态位确认
  ├─ 能力部分覆盖 → 明确补什么
  └─ 能力不匹配 → 缩小生态位范围或放弃重找
```

**决策树二：战略定位 → 选择定位方法（4 问过滤）**

```
第1问：这个品类有明确的领导品牌吗？
  ├─ 没有 → 品类占位型定位
  └─ 有 → 进入第2问

第2问：领导者有没有结构性弱点？（不是"做得不好"，是"做不了"）
  ├─ 有 → 竞争对立型定位
  └─ 没有 → 进入第3问

第3问：品类是否功能同质化严重？
  ├─ 是 → 情感价值型定位
  └─ 否 → 进入第4问

第4问：目标用户是价值观驱动型消费者吗？
  ├─ 是 → 价值主张型定位
  └─ 否 → 回退选择最强路径
```

### 定位策略一致性原则（三条铁律）

1. **定位方法选择主决策依据 = 生态位分析**。用户意见仅在多方案都成立时起 tiebreaker 作用
2. **定位一旦确定，所有下游产出**（使命/愿景/价值观/人格/故事/视觉/广告语/命名）**必须与定位策略保持一致**，不得偏离
3. **用户在下游问题中的回答 = 原材料采集，不是自由选择**。用户说"喜欢爱马仕橙"不意味着品牌应该用橙色——提取的是用户对"品质感"的取向，再由定位策略翻译成具体的视觉方案

### Expand-Confirm 对话模式

每次用户回答后，skill 必须执行三步，不可跳过：
1. **复述理解** — 用你的话复述我对你需求的理解
2. **行业视角拓展** — 从行业规律出发，补充你可能没意识到的关键维度
3. **确认拓展准确性** — 让用户确认补充内容是否准确，纠正后再进入下一轮

这个模式确保信息收集不是"我问你答"的表层交互，而是**顾问式的深度诊断**。

## 理论来源

本 skill 的策略推理建立在以下经典品牌营销理论之上。每个理论在 `references/theory-library.md` 中有完整的"一句话总结→核心逻辑→代表案例→怎么用→在 Skill 中的应用"速查。

| 理论 | 核心主张 | 在 Skill 中的应用 |
|------|---------|-----------------|
| **黄金圈法则** (Simon Sinek) | 从 Why 开始沟通，人们买的是你相信的东西 | 品牌使命/愿景/价值观推导、创始人 IP 叙事框架 |
| **STP 营销理论** (Kotler) | Segmentation→Targeting→Positioning，越精准越有力 | 定位模块的前置逻辑，用户画像的方法论基础 |
| **品牌金字塔** (Millward Brown) | 功能属性→功能利益→情感利益→价值观，逐层上升 | 核心价值梳理的三层结构，四种定位方法的底层原理 |
| **波特五力模型** (Porter) | 行业竞争由五种力量决定 | 竞争格局分析的框架，判断行业进入吸引力的依据 |
| **品牌十二原型** (Mark & Pearson) | 品牌人格可按原型归类，用户认同的是原型而非功能 | 品牌人格化模块的方法论来源 |
| **PESTEL 宏观分析** | 政治/经济/社会/技术/环境/法律六维度扫描外部环境 | 行业趋势分析的维度框架 |
| **SWOT 交叉分析** | S×O（进攻）、W×T（防御）、S×T（转化）、W×O（补强） | 自身诊断的标准操作 |
| **4P/4C 营销组合** | Product/Price/Place/Promotion ↔ Consumer/Cost/Convenience/Communication | 产品策略、定价策略、渠道策略、传播策略的框架 |

> 方法论详情见 `references/methodology.md`，理论速查见 `references/theory-library.md`，输出质量规范见 `references/output-spec.md`。

## 数据引用的方法

本 skill 遵循严谨的数据采集与验证方法论，确保方案的每一条判断都有据可查。

### 五大信息来源

| 来源类型 | 平台/机构 | 用途 |
|---------|---------|------|
| **行业报告平台** | 发现报告、艾瑞咨询、萝卜投研、199IT、洞见研报 | 券商研报、行业深度报告 |
| **证券/财经平台** | 东方财富网、同花顺、雪球 | 上市公司财报、行业板块分析 |
| **社交媒体洞察** | 小红书灵感营销、巨量引擎/巨量算数、快手磁力引擎、腾讯广告 | 用户趋势、品类洞察、消费行为 |
| **第三方研究机构** | Euromonitor、凯度 Kantar、CBNDATA、QuestMobile、麦肯锡/BCG/贝恩 | 行业趋势白皮书、消费报告 |
| **行业协会/组织** | 各行业协会年度报告、年度白皮书、政府/学术机构消费报告 | 行业政策、消费趋势 |

### 交叉验证原则

- 每个关键数据点至少从 **2 类**来源交叉验证
- 券商数据与社媒平台数据对照 — "宏观趋势" vs "微观行为"是否一致
- 第三方机构数据与一线从业者言论对照（访谈、播客、公开演讲）
- 来源有差异时，优先采用**最近发布、样本量更大**的一方
- 不确定的数据标注 **「待调研确认」**，不编造
- 所有引用数据必须标注来源，格式：`[来源: 报告名称/机构, 年份]`

### 信息充分度门槛

brief 模式下，提交的信息需达到 **≥ 7/10** 的充分度评分才进入生成阶段。不足 7 分时，skill 会指出具体缺失项，退回补充后重评。这个门槛确保方案质量不以牺牲信息完整性为代价。

## 三种工作模式

| 模式 | 适合场景 | 工作流程 | 输出 |
|------|---------|---------|------|
| **标准对话模式** | 尚未想清楚，需要顾问引导梳理 | Phase 1 信息收集（6 轮）→ Phase 2 战略定位提问（9 轮）→ Phase 3 执行校验提问（7 轮）→ Phase 4 格式化输出 | HTML / PPTX |
| **brief 文档模式** | 已想清楚、有完整素材，适合 80+ 页深度方案 | 用户填写 Word 问卷 → 3 个 Agent 并行生成（验证→市场研究→策略定位+执行）→ 合并 → 输出 | HTML / PPTX |
| **MD 输出模式** | 大型方案，策略内容与排版代码解耦 | 信息采集 → 策略生成 → 结构化 MD brief → 用户审阅确认 → 单独用排版 skill 转可视化 | 结构化 MD brief |

## 输出质量保障

每份方案必须通过 **8 条质量底线**（信息准确、洞察力强、策略性强、内容有深度、指导意义强、可视化程度高、排版美观、不限制页数）和 **6 项必备要素**（案例佐证、数据来源标注、方法论溯源、可视化成品输出、可执行路线图、风险与假设清单）的检查。

完整质量规范见 `references/output-spec.md`，含四层质量门（Quality Gates）的逐条检核标准。
