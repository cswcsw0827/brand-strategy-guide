# 品牌全案策略指南 (Brand Strategy Guide)

> 基于《品牌全案策略指南 2.0》知识体系的 Claude Code Skill。通过深度对话式诊断，引导你完成从市场调研、战略定位到战略执行的完整品牌全案策略方案。每个关键策略判断必有真实商业案例佐证，最终输出为可直接交付的 HTML 网页 PPT 或 PPTX 文件。

---

## 谁适合用这个工具？

### 初创品牌创始人

作为初创品牌创始人，你面临一个典型的矛盾：

**你有做品牌全案策略的真实需求，但面前横着两道坎：**

1. **不知道怎么做、从哪开始。** 品牌策略是一个系统工程——市场调研、竞争分析、用户洞察、生态位定位、品牌人格、定价策略、传播规划……每一步都需要专业方法论。网上的碎片化文章越看越乱，你缺的不是"知道品牌很重要"的意识，而是一套**完整、可执行的路线图**。

2. **没有足够的预算请品牌咨询公司。** 市面上一次完整的品牌全案咨询，本土中型公司报价通常在 15-50 万，一线国际咨询公司甚至更高。对还在验证产品市场匹配、每一分钱都要算着花的初创团队来说，这笔预算几乎不可能批下来。结果就是：**需求真实存在，但专业服务够不着**。

这个 Skill 正是为解决这个矛盾而生的。它不会给你"老板加油"式的鸡汤文案，而是内置了完整的品牌策略方法论体系（生态位分析、四种定位方法决策树、定价五层塔、四轮驱动传播模型等），通过系统性的追问和方法论推导，帮你从混沌中找到那条**有依据的策略线索**。22 轮深度对话（或直接填写 brief 问卷）之后，你会拿到一份 80-100+ 页的完整品牌全案——拿到就能用、能拿去见投资人。**本质上是把品牌咨询公司的完整工作流，做成了一套 AI 驱动的自助工具。**

### 想要转型品牌策划的设计师

你已经有视觉执行能力，但想往上游走——从"把别人的策略做成好看的图"变成"自己出策略"。这个 skill 内置的完整方法论体系（生态位分析、四种定位方法决策树、定价五层塔、四轮驱动传播模型等）可以成为你的**策略思维训练场**。每次使用都是一次跟着框架走一遍完整策略推导的机会，方法论和理论库会告诉你每一步背后的"为什么"。

### 准备创业做品牌的人

你看到了一个市场机会，但需要系统性地验证它是否成立——行业是否值得进入、竞争格局有没有缺口、用户有没有未被满足的真实需求、你的团队能不能吃得下。这个 skill 的市场调研模块会带你完成这四个步骤的交叉验证，**找到你的生态位**——那个市场空白、用户需求、自身能力三者交汇的位置。如果生态位不存在，skill 会如实告诉你，而不是编一个给你。

## 方法论体系

### 核心底层逻辑：生态位驱动的价值三段论

整个方法论体系建立在一个基本命题之上：**品牌策略的核心问题不是"怎么卖更多"，而是"我在生态中占据什么位置"。**

```
发现价值（市场调研）  →  创造价值（战略定位）  →  传播价值（战略执行）
      ↓                                              ↓
  找到生态位                                     商业化交付
      ↓                                              ↓
      └─────────── 循环反馈，持续校准 ←─────────────┘
```

三个阶段不是线性步骤，而是一个**价值正向循环**：

- **市场调研**的最终目的不是堆积数据，而是帮品牌找到**生态位**——市场中一个未被充分满足、且自身有能力占据的位置
- **战略定位**是基于生态位来构建品牌的身份——"我是谁"必须回答"我在生态中占据什么位置"
- **战略执行**是将生态位转化为可感知的用户价值，通过产品和传播让生态位从分析结论变成用户心智中的事实

生态位 = 市场空白 ∩ 用户需求 ∩ 自身能力。生态位找到之后，品牌才有清晰的"不做什么"的边界。

### 两个核心决策树

一切策略判断由两个不可跳过的决策树驱动：

**决策树一：市场调研 → 找到生态位（4 步漏斗）**

```
第1步：行业是否值得进入？
  ├─ 大市场+高增长+政策利好 → 继续
  ├─ 中等市场 → 谨慎进入（需强理由）
  └─ 小市场+萎缩 → 换赛道或等结构性窗口

第2步：竞争格局有没有缺口？
  ├─ CR5<30%（分散市场）→ 品类占位机会
  ├─ CR5 30-60%（集中化中）→ 找细分切口
  └─ CR5>60%（寡头市场）→ 只能走对立或边缘颠覆

第3步：用户有没有未被满足的需求？
  ├─ 多画像共有痛点+竞品无人解决 → 强生态位信号（P0）
  ├─ 部分画像有痛点+竞品已在解决 → 弱生态位信号（P1）
  └─ 无明显未满足需求 → 需差异化定位

第4步：我能不能吃得下？
  ├─ 能力完全覆盖 → 生态位确认
  ├─ 能力部分覆盖 → 明确补什么
  └─ 能力不匹配 → 缩小生态位范围或放弃重找
```

**决策树二：战略定位 → 选择定位方法（4 问过滤）**

```
第1问：这个品类有明确的领导品牌吗？
  ├─ 没有 → 品类占位型定位
  └─ 有 → 进入第2问

第2问：领导者有没有结构性弱点？（不是"做得不好"，是"做不了"）
  ├─ 有 → 竞争对立型定位
  └─ 没有 → 进入第3问

第3问：品类是否功能同质化严重？
  ├─ 是 → 情感价值型定位
  └─ 否 → 进入第4问

第4问：目标用户是价值观驱动型消费者吗？
  ├─ 是 → 价值主张型定位
  └─ 否 → 回退选择最强路径
```

### 定位策略一致性原则（三条铁律）

1. **定位方法选择主决策依据 = 生态位分析**。用户意见仅在多方案都成立时起 tiebreaker 作用
2. **定位一旦确定，所有下游产出**（使命/愿景/价值观/人格/故事/视觉/广告语/命名）**必须与定位策略保持一致**，不得偏离
3. **用户在下游问题中的回答 = 原材料采集，不是自由选择**。用户说"喜欢爱马仕橙"不意味着品牌应该用橙色——提取的是用户对"品质感"的取向，再由定位策略翻译成具体的视觉方案

### Expand-Confirm 对话模式

每次用户回答后，skill 必须执行三步，不可跳过：
1. **复述理解** — 用你的话复述我对你需求的理解
2. **行业视角拓展** — 从行业规律出发，补充你可能没意识到的关键维度
3. **确认拓展准确性** — 让用户确认补充内容是否准确，纠正后再进入下一轮

这个模式确保信息收集不是"我问你答"的表层交互，而是**顾问式的深度诊断**。

## 理论来源

本 skill 的策略推理建立在以下经典品牌营销理论之上。每个理论在 `references/theory-library.md` 中有完整的"一句话总结→核心逻辑→代表案例→怎么用→在 Skill 中的应用"速查。

| 理论 | 核心主张 | 在 Skill 中的应用 |
|------|---------|-----------------|
| **黄金圈法则** (Simon Sinek) | 从 Why 开始沟通，人们买的是你相信的东西 | 品牌使命/愿景/价值观推导、创始人 IP 叙事框架 |
| **STP 营销理论** (Kotler) | Segmentation→Targeting→Positioning，越精准越有力 | 定位模块的前置逻辑，用户画像的方法论基础 |
| **品牌金字塔** (Millward Brown) | 功能属性→功能利益→情感利益→价值观，逐层上升 | 核心价值梳理的三层结构，四种定位方法的底层原理 |
| **波特五力模型** (Porter) | 行业竞争由五种力量决定 | 竞争格局分析的框架，判断行业进入吸引力的依据 |
| **品牌十二原型** (Mark & Pearson) | 品牌人格可按原型归类，用户认同的是原型而非功能 | 品牌人格化模块的方法论来源 |
| **PESTEL 宏观分析** | 政治/经济/社会/技术/环境/法律六维度扫描外部环境 | 行业趋势分析的维度框架 |
| **SWOT 交叉分析** | S×O（进攻）、W×T（防御）、S×T（转化）、W×O（补强） | 自身诊断的标准操作 |
| **4P/4C 营销组合** | Product/Price/Place/Promotion ↔ Consumer/Cost/Convenience/Communication | 产品策略、定价策略、渠道策略、传播策略的框架 |

## 数据引用的方法

本 skill 遵循严谨的数据采集与验证方法论，确保方案的每一条判断都有据可查。

### 五大信息来源

| 来源类型 | 平台/机构 | 用途 |
|---------|---------|------|
| **行业报告平台** | 发现报告、艾瑞咨询、萝卜投研、199IT、洞见研报 | 券商研报、行业深度报告 |
| **证券/财经平台** | 东方财富网、同花顺、雪球 | 上市公司财报、行业板块分析 |
| **社交媒体洞察** | 小红书灵感营销、巨量引擎/巨量算数、快手磁力引擎、腾讯广告 | 用户趋势、品类洞察、消费行为 |
| **第三方研究机构** | Euromonitor、凯度 Kantar、CBNDATA、QuestMobile、麦肯锡/BCG/贝恩 | 行业趋势白皮书、消费报告 |
| **行业协会/组织** | 各行业协会年度报告、年度白皮书、政府/学术机构消费报告 | 行业政策、消费趋势 |

### 交叉验证原则

- 每个关键数据点至少从 **2 类**来源交叉验证
- 券商数据与社媒平台数据对照 — "宏观趋势" vs "微观行为"是否一致
- 第三方机构数据与一线从业者言论对照（访谈、播客、公开演讲）
- 来源有差异时，优先采用**最近发布、样本量更大**的一方
- 不确定的数据标注 **「待调研确认」**，不编造
- 所有引用数据必须标注来源，格式：`[来源: 报告名称/机构, 年份]`

### 信息充分度门槛

brief 模式下，提交的信息需达到 **≥ 7/10** 的充分度评分才进入生成阶段。不足 7 分时，skill 会指出具体缺失项，退回补充后重评。这个门槛确保方案质量不以牺牲信息完整性为代价。

## 三种工作模式

> 品牌全案策略指南提供三种工作模式，覆盖从「完全没想法」到「已有完整素材」再到「超大方案需解耦」的全场景。选择哪种模式，取决于你的准备程度和方案规模。

---

### 模式一：标准对话模式 — 深度诊断型

> "你不确定你的品牌应该是什么——我来帮你问出来。"

最完整、最深度的模式。通过 22 轮结构化对话，像一位资深品牌顾问一样，一步步引导你理清品牌全貌。

**工作流程（4 阶段、22 轮对话）：**

| 阶段 | 轮次 | 内容 |
|------|------|------|
| Phase 1A 信息收集 | 6 轮 | 行业赛道、竞品格局、目标用户、产品特性、团队能力、商业目标 |
| Phase 1B 交叉验证 | — | 多源数据交叉验证，标注不确定项 |
| Phase 2 战略定位提问 | 9 轮 | 使命、愿景、价值观、品牌人格、品牌故事、视觉方向、广告语、命名 |
| Phase 3 执行校验提问 | 7 轮 | 产品策略、定价策略、渠道策略、内容策略、社群策略、预算规划、团队配置 |
| Phase 4 格式化输出 | — | 调用 guizang-ppt-skill 或 ppt-master 生成 HTML / PPTX |

**🎯 适用场景：**
- 创业早期，对品牌方向还不清晰
- 需要外部视角帮助梳理和挑战假设
- 享受深度对话过程，愿意花时间挖出洞察

**✅ 核心优势：** Expand-Confirm 对话模式（复述理解 → 行业拓展 → 确认准确性）能挖出创始人自己都没想到的洞察。每个关键策略判断必有真实商业案例佐证。

**⚠️ 潜在局限：** 22 轮对话耗时长（约 2-3 小时）；超大方案（100+ 页）可能在上下文窗口遭遇瓶颈。

---

### 模式二：Brief 文档模式 — 高效并行型

> "你已经想清楚了——直接填问卷，3 个 Agent 并行出方案。"

跳过 22 轮对话，用户通过填写结构化 Word 问卷提交完整信息，3 个 AI Agent 并行工作，快速生成深度方案。

**工作流程：**

```
你填写 Word 问卷（8 章结构化表单）
        ↓
Skill 读取并提取结构化信息
        ↓
Agent A：验证信息充分度（≥ 7/10 才继续）
        ↓
┌───────────── 并行生成 ─────────────┐
│  Agent B              Agent C       │
│  市场调研              战略定位+执行  │
│  · 外部环境扫描        · 品牌体系    │
│  · 竞争格局分析        · 产品策略    │
│  · 用户洞察            · 定价+传播   │
│                       · 商业化路线图 │
└───────────── 结果合并 ─────────────┘
        ↓
格式化输出（HTML / PPTX / MD brief）
```

**🎯 适用场景：**
- 品牌方向已基本清晰，有完整素材
- 需要 80+ 页深度方案，但不想花 2-3 小时对话
- 团队已做过前期调研，有数据可填入问卷

**✅ 核心优势：** 3 Agent 并行生成，耗时约 30-60 分钟；Word 问卷格式用户友好，可离线填写后发给团队协作；信息一致性高（所有 Agent 基于同一份 brief）。

**⚠️ 前提条件：** 问卷信息充分度必须 ≥ 7/10，不足则退回标准对话模式补信息。

**📋 问卷文件：** `output/brief_questionnaire.docx` — 可直接发送给客户或团队成员填写。模板参考 `instructions/brief_template.md`，填写示例见 `instructions/brief_sample.md`。

---

### 模式三：MD 输出模式 — 内容排版解耦型

> "方案太大，策略内容和排版代码不能挤在同一个上下文里——分开做，各司其职。"

专为 80-100+ 页大型方案设计。策略内容与排版代码完全分离：skill 专注策略推导并产出结构化 MD brief，用户审阅确认后，再单独调用排版 skill 做可视化。

**工作流程：**

```
信息采集（对话 / brief 均可）
        ↓
策略内容生成（全案推导）
        ↓
交付结构化 MD brief（精确到每页标题 / 数据 / 图表 / 案例）
        ↓
用户审阅确认策略内容
        ↓
单独使用 guizang-ppt-skill 或 ppt-master
基于 MD brief 生成可视化成品（HTML / PPTX）
```

**为什么需要这个模式？**

当方案达到 80-100+ 页时，如果策略内容（文本、数据、案例）和排版代码（CSS/JS/HTML 模板）在同一个上下文窗口，会互相挤占——要么策略深度不够，要么排版质量下降。MD 输出模式将两者解耦：
- **策略生成阶段**：上下文 100% 用于策略推理，产出高精度结构化 MD
- **排版阶段**：排版 skill 的上下文 100% 用于视觉设计，不浪费在理解策略上

**🎯 适用场景：**
- 80-100+ 页大型品牌全案
- 对策略深度和排版质量都有极高要求
- 策略内容可能需要多轮迭代（改 MD 秒级，不用重跑整个 pipeline）

**✅ 核心优势：** 策略迭代快（改 MD 秒级）；上下文不膨胀，策略质量和排版质量都能拉满；MD 可复用为官网文案、摘要、投资人材料。

**📌 硬约束：** MD 必须写到可直接排版的精度——不是散文，而是每页精确标注标题层级、数据点位、图表类型、案例段落。禁止输出段落堆砌、无页级结构的散文式 MD。详见 `references/output-spec.md` 第十九章。

## 输出质量保障

每份方案必须通过 **8 条质量底线**（信息准确、洞察力强、策略性强、内容有深度、指导意义强、可视化程度高、排版美观、不限制页数）和 **6 项必备要素**（案例佐证、数据来源标注、方法论溯源、可视化成品输出、可执行路线图、风险与假设清单）的检查。

```bash
git clone https://github.com/DevinKuang/brand-strategy-guide.git ~/.claude/skills/brand-strategy-guide
```

克隆后无需额外配置，Claude Code 会自动加载 `SKILL.md`。

---

## 依赖

### 输出 Skill（至少需要一个）

| Skill | 用途 | 输出 |
|-------|------|------|
| `guizang-ppt-skill` | HTML 网页 PPT（横向翻页，WebGL 背景） | HTML |
| `ppt-master` | AI 多角色协作生成 SVG → PPTX | PPTX |

> 如果以上两个 skill 都未安装，只能走 **MD 输出模式**——生成结构化 Markdown brief 后，单独交给排版工具处理。

### Python 工具（仅 brief 问卷转换）

`scripts/convert_brief_to_docx.py` 将 Markdown 格式的 brief 问卷转为 Word 文档（方便发给客户填写）：

```bash
pip install python-docx
python scripts/convert_brief_to_docx.py input.md [output.docx]
```

## 目录结构

```
brand-strategy-guide/
├── SKILL.md                     # 主 skill 定义（三种模式 + 完整对话流程）
├── instructions/                # Agent 指令
│   ├── agent_market.md          #   市场调研 agent（外部扫描+竞品+用户洞察）
│   ├── agent_positioning.md     #   战略定位 agent（品牌体系构建）
│   ├── agent_execution.md       #   战略执行 agent（产品+定价+传播+商业化）
│   ├── brief_template.md        #   brief 问卷模板（结构化表单）
│   └── brief_sample.md          #   brief 问卷填写示例
├── references/                  # 方法论参考
│   ├── system-overview.md       #   系统全景图（底层逻辑+内容框架+输出标准）
│   ├── output-spec.md           #   输出质量规范（8 原则+6 必备+4 层质量门）
│   ├── methodology.md           #   方法论详解（市场调研+定位+执行全流程）
│   └── theory-library.md        #   理论速查库（8 个经典营销理论）
└── scripts/
    └── convert_brief_to_docx.py # MD → DOCX 转换工具
```
