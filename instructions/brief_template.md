## 9. 验证与纠偏机制

> **核心原则**: brief.md 是甲方自述，天然包含乐观偏差、过时数据、认知盲区。
> Agent 不可全盘采信，必须在生成前完成交叉验证。验证结果写入 `audit_report.md`。

### 9.1 信息可信度分级

| 等级 | 标记 | 定义 | 处理方式 | 典型示例 |
|------|------|------|----------|----------|
| **S** | 客观可验证 | 有公开数据可交叉验证的客观事实 | 必须用 2+ 来源验证，不一致时报错 | 市场规模、竞品价格、法规 |
| **A** | 可挑战 | 甲方主观判断，但有外部信号可佐证或反驳 | 搜索反例，若发现矛盾则标注并给出修正建议 | “产品力行业第一”、”没有直接竞品” |
| **B** | 待补充 | brief.md 标注”未知”或留空的字段 | Agent 主动搜索填充，标注数据来源和可信度 | 空白的市场规模、缺失的竞品信息 |
| **C** | 不可验证 | 纯主观、无外部信号可验证的信息 | 原样采纳但标注”未验证”，不作为唯一决策依据 | 创始人动机、审美偏好 |

### 9.2 各 Agent 的验证职责

| Agent | 验证对象 | 验证方式 | 必须搜索的关键词 |
|-------|----------|----------|------------------|
| **Agent A** | §1 行业与市场 | 搜索行业报告、财经数据、政策文件，交叉验证规模/CAGR/PESTEL 信号 | “[行业名] 市场规模 2024 2025”、”[行业名] 政策 最新” |
| | §2 竞争格局 | 搜索每个竞品的最新动态、真实价格、用户评价，补充遗漏竞品 | “[竞品名] 融资/新品/负面”、”[品类] 品牌排行 2025” |
| | §3 用户画像 | 搜索社交媒体上的真实用户讨论，对比画像中的痛点是否真实高频 | “[品类] 吐槽/避雷/推荐 小红书”、”[品类] 用户需求 调研” |

### 9.3 矛盾处理规则

当 Agent 验证发现 brief.md 与外部数据不一致时：

| 冲突类型 | 处理规则 |
|----------|----------|
| **数据冲突**（如 brief 说市场 100 亿，报告显示 50 亿）| 优先采信外部数据（标注来源），在 audit_report 中标注”brief.md 数据偏高/偏低 X%”，正文采用修正后数据 |
| **判断冲突**（如 brief 说”没有竞品”，但搜索发现 3 家类似品牌）| 补充遗漏竞品，在 audit_report 中标注”甲方认知盲区”，正文以修正后的竞争格局为准 |
| **自评偏高**（如六维诊断自评全部 7+ 分，但外部口碑一般）| 下调至合理区间，附外部证据（用户评论截图/评分），标注”自评偏高 X 分” |
| **预算异常**（如预算远低于行业 benchmark）| 不直接否定，而是标注”建议预算为 X 万（当前 Y 万），差距 Z%。如需坚持当前预算，建议优先聚焦以下 3 个动作...” |
| **不可调和冲突**（双方数据都有来源、无法判断谁对）| 两条线都保留在正文中，用”争议标注”框展示，让甲方自行判断 |

## 模板使用说明

### 填写流程
1. **粗填**（30 min）：把所有能填的字段快速填上，不确定的标”未知”
2. **深挖**（60 min）：回到标”未知”的字段，通过搜索/访谈/内部讨论补充
3. **自评**（15 min）：完成第 7 节的自评，判断是否达到 70% 阈值
4. **提交**：达到阈值后，进入 Session 2（Agent 生成）

### 关键填写原则
- **”未知”优于瞎填**: 明确的”未知”比模糊的猜测对 Agent 更有价值——Agent 会知道哪里需要搜索补充
- **具体优于抽象**: “25-35 岁一线城市白领女性”不如”28 岁上海静安区产品经理小林，月入 25K，每月在护肤品上花 1500”
- **口语优于书面**: 用户原话、创始人动机用真实口语，不要润色成书面语
- **数量优于质量**（在列举阶段）: 竞品能列 10 家就不列 5 家，画像能写 3 个就不写 2 个
