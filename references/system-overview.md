# 品牌全案策略生成器 — 系统全景图 v2.0

> 本文档描述 skill 的完整架构：底层逻辑 → 内容框架 → 输出标准 → 文件框架。
> 每次执行前用于对齐理解，每次修改后用于校验一致性。

---

## 一、底层逻辑

### 1.1 价值三段论（核心循环）

品牌策略的核心问题不是"怎么卖更多"，而是"我在生态中占据什么位置"。三个阶段不是线性步骤，而是一个价值正向循环：

```
发现价值（市场调研）  →  创造价值（战略定位）  →  传播价值（战略执行）
      ↓                                              ↓
  找到生态位                                     商业化交付
      ↓                                              ↓
      └─────────── 循环反馈，持续校准 ←─────────────┘
```

**生态位是全部工作的起点和终点**：

- 市场调研的最终目的不是堆积数据，而是帮品牌找到生态位——市场中一个未被充分满足、且自身有能力占据的位置
- 战略定位是基于生态位来构建品牌的身份——"我是谁"必须回答"我在生态中占据什么位置"
- 战略执行是将生态位转化为可感知的用户价值——让生态位从分析结论变成用户心智中的事实
- 生态位找到之后，品牌才有清晰的"不做什么"的边界

### 1.2 两个核心决策树

一切策略判断由两个决策树驱动，不可跳过。

**决策树一：市场调研 → 找到生态位（4步漏斗）**

```
第1步：行业是否值得进入？
  ├─ 大市场+高增长+政策利好 → 继续
  ├─ 中等市场 → 谨慎进入（需强理由）
  └─ 小市场+萎缩 → 换赛道或等结构性窗口
        │
第2步：竞争格局有没有缺口？
  ├─ CR5<30%（分散市场）→ 品类占位机会
  ├─ CR5 30-60%（集中化中）→ 找细分切口
  └─ CR5>60%（寡头市场）→ 只能走对立或边缘颠覆
        │
第3步：用户有没有未被满足的需求？
  ├─ 多画像共有痛点+竞品无人解决 → 强生态位信号（P0）
  ├─ 部分画像有痛点+竞品已在解决 → 弱生态位信号（P1）
  └─ 无明显未满足需求 → 需差异化定位
        │
第4步：我能不能吃得下？
  ├─ 能力完全覆盖 → 生态位确认
  ├─ 能力部分覆盖 → 明确补什么
  └─ 能力不匹配 → 缩小生态位范围 或 放弃重找
        │
        ▼
  生态位 = 市场空白 ∩ 用户需求 ∩ 自身能力
```

**决策树二：战略定位 → 选择定位方法（4问过滤）**

```
第1问：这个品类有明确的领导品牌吗？
  ├─ 没有 → 🗝️ 品类占位型（切割·定义品类）
  └─ 有 → 进入第2问
        │
第2问：领导者有没有结构性弱点？（不是"做得不好"，是"做不了"）
  ├─ 有 → 🗝️ 竞争对立型（对立·你做不到我做）
  └─ 没有/不确定 → 进入第3问
        │
第3问：品类是否功能同质化严重？
  ├─ 是 → 🗝️ 情感价值型（情绪·绑定情感场景）
  └─ 否 → 进入第4问
        │
第4问：目标用户是价值观驱动型消费者吗？
  ├─ 是 → 🗝️ 价值主张型（意义·宣告信念）
  └─ 否 → 回退到第2或第3步，选最强路径
```

**两个决策树的联动**：市场调研的每个产出都限制定位方法的选择范围。如果定位方法与生态位不匹配，回到市场调研重新校准。

### 1.3 定位策略一致性原则（三条铁律）

定位策略是所有下游决策的主轴。用户回答是原材料，不是选择权。

**铁律一**：定位方法选择主决策依据 = 生态位分析。用户意见仅在多方案都成立时起 tiebreaker 作用。
**铁律二**：定位一旦确定，所有下游产出（使命/愿景/价值观/人格/故事/视觉/广告语/命名）必须与定位策略保持一致。
**铁律三**：用户在下游问题中的回答 = 原材料采集。通过定位策略过滤器加工后才产出最终内容。

```
用户回答 → ┬─ 与定位方向一致 → ✅ 直接采用
           ├─ 部分相关但有偏差 → ⚠️ 提取有用部分，重新加工
           └─ 与定位方向矛盾 → ❌ 放弃，解释原因
```

### 1.4 Expand-Confirm 对话模式

每次用户回答后，必须执行三步，不可跳过：

```
用户回答 → ① 复述理解 → ② 行业视角拓展 → ③ 确认 → 进入下一轮
```

应用于 Phase 1A（6轮）、Phase 3A（9轮）、Phase 3B（7轮），共 22 轮对话中每轮必须执行。

## 二、内容框架

### 2.1 Phase 全景图

```
Phase 1：信息收集（4个子阶段）
  ├─ 1A：用户对话（6轮深度对话）
  ├─ 1B：网络数据采集（5类来源 × 9个搜索维度）
  ├─ 1C：交叉验证（假设验证 + 信息充分度评分 0-10）
  └─ 1D：竞品扩展（从3个种子到10家全景）
        │
        ▼  ⛔ Layer 0 Gate: 信息充分度 ≥ 7.0/10
        │
Phase 2：进入策略生成（过渡检查点）
        │
        ▼
Phase 3：策略生成（3个子阶段）
  ├─ 3A：战略定位对话（9轮对话）
  │     ⛔ Layer 1 Gate: 五要素一致性（定位+使命+愿景+价值观+人格）
  ├─ 3B：执行验证对话（7轮对话 + 早期三原则 + 策略前提总结）
  └─ 3C：内容生成（32个模块，逐页深度门 Layer 2 + 完整性检查 Layer 3）
        │
        ▼
Phase 4：格式化输出（PPTX 或 HTML，严禁 MD）
```

### 2.2 各 Phase 详细内容

**Phase 1A — 6 轮信息收集**

| 轮次 | 收集内容 | 追问深度 |
|------|---------|---------|
| 1 | 行业/品类 + 产品描述 + 创始人动机 | 3个追问方向 |
| 2 | 目标用户画像 + 使用场景 | 人口统计/行为/情绪/替代方案/场景 × 3步追问 |
| 3 | 竞品（2-3个种子）+ 品牌阶段 + 品牌名称 | 品牌名深度追问（来源/含义/用户反馈/商标/替代名） |
| 4 | 团队优势 + 资源/供应链能力 | 核心能力/缺口/过往案例 × 3步追问 |
| 5 | 预算量级 + 资源优先级 | 资金来源/额度/分配倾向/已有资产 × 3步追问 |
| 6 | 综合确认 + 缺口补充 | 总结前5轮，确认理解，补充遗漏 |

**Phase 1B — 网络数据采集（5类来源）**

- 行业报告平台（艾瑞/易观/QuestMobile等）
- 证券/财经平台（东方财富/同花顺/36氪等）
- 社媒平台洞察（小红书/抖音/微博趋势）
- 权威第三方研究（学术论文/NGO报告/政府统计）
- 行业协会报告

**Phase 1C — 交叉验证 + 信息充分度评分**

7 维度加权评分：市场数据 25% / 用户画像 20% / 竞品信息 20% / 团队能力 15% / 预算资源 10% / 品牌名称资产 5% / 创始人动机 5%

≥ 7.0 → 进入 Phase 3；5.0-6.9 → 标注弱项可进入；< 5.0 → 回到 Phase 1A

**Phase 3A — 9 轮战略定位对话**

| 轮次 | 内容 | 产出的模块 |
|------|------|----------|
| 1 | 定位方向确认（方法选择） | 模块 9：定位方法 |
| 2 | 品牌使命 | 模块 11：使命 |
| 3 | 品牌愿景 | 模块 11：愿景 |
| 4 | 品牌价值观 | 模块 11：价值观 |
| 5 | 品牌故事（我心→我志→我诺） | 模块 13：品牌故事 |
| 6 | 品牌人格（12原型方法论） | 模块 12：品牌人格 |
| 7 | 品牌视觉符号（视觉翻译+审美参考） | 模块 16：视觉系统 |
| 8 | 广告语偏好 | 模块 15：广告语 |
| 9 | 品牌命名（已有→诊断/无→创建） | 模块 14：命名 |

**Phase 3B — 7 轮执行验证对话**

| 轮次 | 内容 |
|------|------|
| 1 | 研发能力 |
| 2 | 供应链/生产能力 |
| 3 | 内容能力 |
| 4 | 创始人出镜意愿 |
| 5 | 渠道/销售能力 |
| 6 | 团队规模和结构 |
| 7 | 资源优先级 |

**Phase 3C — 内容生成（模块化结构，页数由内容深度驱动）**

内容按模块组织，每个模块的页数由信息密度和深度要求决定，不预设上限。见下文"四、输出文件框架"。

## 三、输出标准和约束

> **核心标准（五条根原则）**：深度优先 → 全面覆盖 → 洞察驱动 → 图表与数据佐证 → 策略逻辑自洽。页数是内容深度和全面性的自然结果，不是预设目标。详见 output-spec.md 第〇章。

完整规范见 `references/output-spec.md`（18章）。以下为体系概要。

### 3.1 四层质量门

| 层级 | 时机 | 检查内容 | 不通过处理 |
|------|------|---------|----------|
| **Layer 0** | Phase 1C 完成后 | 信息充分度 ≥ 7.0/10 | 回到 Phase 1A 补充提问 |
| **Layer 1** | Phase 3A 完成后 | 定位五要素无矛盾 | 回到矛盾模块重新校准 |
| **Layer 2** | Phase 3C 每模块生成后 | 深度 ≥ L3（洞察） | 重写当前模块 |
| **Layer 3** | Phase 3C 全部完成后 | 40+ 项完整性清单 | 逐项修复后重新检查 |

### 3.2 深度标准 L1-L4

| 级别 | 标准 | 是否合格 |
|------|------|---------|
| L1 罗列 | 信息被列出，换个品牌也能用 | ❌ |
| L2 分析 | 有"因为…所以…"因果链 | ❌ |
| **L3 洞察** | 提炼出非显而易见的结论，读者"原来如此" | ✅ 最低合格线 |
| **L4 策略** | 结论转化为可执行行动，创始人明天能做 | ✅ |

### 3.3 输出八原则

准确 → 洞察 → 策略 → 深度 → 指导 → 可视化 → 排版 → 不限页数

### 3.4 六项必备要素

案例佐证 / 数据来源标注 / 方法论溯源 / 可视化成品输出 / 可执行路线图 / 风险与假设清单

### 3.5 逐页最低标准 + 图片-策略一致性

每个内容模块有具体硬性要求（数据/图表/深度），5个内容模块（用户画像/品牌人格/品牌故事/视觉识别/产品策略）有AI配图强制要求，每张图从策略结论中派生 prompt（详见 output-spec.md 第七、八章）。

### 3.6 自检-修正循环

每模块生成后立即三步自检：
1. "读者会说'所以呢'还是'原来如此'？"
2. "这个分析换到竞品身上也成立吗？"
3. "至少有[数据/案例/图表]中的两项支撑吗？"

## 四、输出文件框架

### 4.1 内容模块结构（页数由内容深度决定，不预设上限）

以下为**必须覆盖的内容模块**。每个模块至少 1 页；数据密集或需案例展开时，一个模块可以拆为多页。页数不是目标，内容完整和深度才是。

```
封面      1+ 页   品牌名 + 核心主张 + 氛围背景图
目录      1+ 页   四部分概览

Part 1: 发现价值（市场调研）
  分隔页  1 页
  PESTEL 宏观环境分析        1-2 页   ← 6根柱子，数据多时可每柱独立
  竞争格局全景（10家竞品）    1-3 页   ← 矩阵+气泡图+关键发现，可拆
  结构性缺口（2×2定位图）    1 页
  🖼 用户画像                4+ 页   ← 4个画像，每人可独立1页（含场景图）
  消费者旅程（情绪曲线）      1-2 页   ← 曲线+AISAS矩阵，可拆
  SWOT + 生态位结论          1-2 页   ← SWOT本身+TOWS交叉，可拆

Part 2: 创造价值（战略定位 —— 资深品牌策略总监视角）
  分隔页  1 页
  定位方法决策（四问决策树）  1-2 页   ← 每一步标注"为什么是这个分支"+同方法案例+为什么不用其他三种
  品牌定位语                 1 页     ← 填表版+压缩版+竞品定位并排对比+三把尺深度分析
  品牌价值画布               1-2 页   ← 用户侧三层+品牌侧价值主张+溯源到生态位
  使命·愿景·价值观           1-3 页   ← 使命有"为什么"+价值观有行为定义和反例+愿景分阶段+标杆案例
  🖼 品牌人格化              1-2 页   ← 原型轮盘+一句话概括展开+同原型标杆+调性卡片+原型视觉化图
  🖼 品牌故事                1-3 页   ← 三幕叙事(我心→我志→我诺)+场景细节+用户洞察融入+场景插画
  命名 & 广告语              1-2 页   ← 命名6维诊断+广告语场景模拟+五把尺分析+广告语策略案例
  🖼 视觉识别                1-3 页   ← 色彩板(含选择理由)+字体(含气质描述)+否定清单(含原因)+风格拼贴板

> Part 2 的核心不是"呈现信息"，而是"创造价值"。写这部分时，角色是资深品牌策略总监——有自己的观点、有方法论的自信、有案例的支撑。同时站在用户角度理解需求（把洞察转化为品牌价值点）和品牌角度进行价值创造（品牌主动选择，不是被动满足）。

Part 3: 传播价值（战略执行）
  分隔页  1 页
  🖼 产品策略 & MVP          1-3 页   ← Must/Can/Won't + 路线图 + 产品概念图
  三层价值金字塔             1-2 页   ← 金字塔+竞品对比
  定价策略                   1-2 页   ← 成本结构+竞品锚点+定价铁律+案例
  创始人IP策略               1-2 页   ← 三个故事+内容节奏
  内容营销策略               1-3 页   ← AISAS矩阵+内容类型+四不原则
  社群建设                   1-2 页   ← 定位+冷启动路径+运营节奏
  渠道策略 & 年度规划        2-4 页   ← 甘特图+预算+KPI+风险+团队

附录
  理论溯源                   1-3 页   ← 7个框架可各独立展示
  完整性检查 & 待验证假设    1-2 页
  巨人的肩膀                 1 页

感谢页      1 页
```

**页数弹性范围**：基准约 25-30 页，深度展开可达 40-50 页。下限由内容完整性约束，上限由信息密度约束——不因减少页数而牺牲深度，也不为凑页数而注水。

### 4.2 内容模块类型分类

| 类型 | 对应模块 | 内容形式 | 是否有AI图 |
|------|---------|---------|----------|
| **Hero 页** | 封面、三部分分隔页、感谢页 | 全幅背景图 + 标题/感谢文字 | 是（氛围背景） |
| **数据图表页** | PESTEL、竞品全景、结构性缺口、SWOT | SVG 矢量图表 + 数据标注 | 否 |
| **用户分析页** | 用户画像、消费者旅程 | 画像卡片 + 旅程曲线 | 用户画像是（每人1张场景图） |
| **策略框架页** | 定位决策树、价值画布、价值金字塔 | SVG 框架图 + 推导说明 | 否 |
| **品牌定义页** | 定位语、使命愿景价值观、人格、故事、命名广告语、视觉识别 | 文字 + 框架 + 视觉元素 | 人格/故事/视觉识别是 |
| **执行计划页** | 产品策略、定价、创始人IP、内容、社群、年度规划 | 框架 + 表格 + 甘特图 | 产品策略是 |
| **引用附录页** | 理论溯源、检查清单、巨人启示 | 框架图 + 表格 | 否 |
