## 1. 黄金圈法则 — 从 Why 开始

**一句话总结**：人们买的不是你做的产品，是你相信的东西。从 Why（为什么做）→ How（怎么做）→ What（做什么），由内而外沟通。

**核心逻辑**：
- 大多数品牌从外到内沟通：先说"我们做什么"，最后才说"为什么"
- 伟大品牌从内到外：先说"我们相信什么"，再说"所以我们怎么做"，最后"所以我们做了这个产品"
- 大脑边缘系统（情感/决策）被 Why 触动，新皮层（理性/语言）被 What 触动——但决策由边缘系统驱动

**代表案例**：苹果 — "我们挑战现状（Why），我们相信不同的思考方式（How），所以我们做出了设计精美的电脑（What）" vs 普通电脑公司 — "我们做最棒的电脑（What），设计精美使用简单（How），买一台吗？"

**怎么用**：
1. Why：你为什么存在？你相信什么？（不是"为了赚钱"）
2. How：你如何实现这个信念？你的方法和别人有什么不同？
3. What：你具体做什么产品？

**在 Skill 中的应用**：
- 模块14（品牌使命/愿景/价值观）：Why 推导的底层逻辑
- 模块22（创始人IP）：创始人叙事框架的"冲突→顿悟→方案"就是黄金圈的叙事版
- 模块16（品牌故事）：黄金圈法则是品牌故事的核心结构

## 2. STP营销理论 — 别试图讨好所有人

**一句话总结**：把市场切细（Segmentation）→ 选择最有价值的细分（Targeting）→ 在用户脑子里占一个独特位置（Positioning）。越精准越有力。

**核心逻辑**：
- 试图讨好所有人 = 谁也讨好不了
- 先切分市场（按人口/地理/心理/行为维度），再选择目标（看规模/成长性/竞争强度/匹配度），最后建立定位（和竞品有什么不同？这个不同对用户重要吗？）

**代表案例**：lululemon — 选择了"女性瑜伽爱好者"这个细分（Segmentation），这群人在增长、耐克阿迪没有专门针对瑜伽的产品（Targeting），定位为"瑜伽生活方式品牌"而非"卖瑜伽裤的"（Positioning）

**怎么用**：
1. 把大市场切成若干小块，每一块有相似需求
2. 选择最有吸引力、最匹配你能力的细分市场
3. 在目标用户的脑子里，占一个和竞品不同的位置

**在 Skill 中的应用**：
- 模块9（品牌定位策略）：STP是定位的前置逻辑——没有S和T，P就是拍脑袋
- 模块4（用户画像）：用户画像是Segmentation的产出物
- 模块8（四种定位方法）：每种定位方法都建立在特定的Targeting逻辑上

## 3. 品牌金字塔 — 你的品牌到底在卖什么？

**一句话总结**：品牌从功能属性→功能利益→情感利益→价值观，一层一层往上搭。大部分人只做到了第一层，高手做到了第三、四层。

**核心逻辑**：
- 底层（功能属性）：产品有什么功能？→ 竞争壁垒最低，最容易陷入参数战
- 二层（功能利益）：用户得到什么？→ 从"我有什么"切换到"你能得到什么"
- 三层（情感利益）：用户感受到什么？→ 脱离产品本身，进入心理感受
- 顶层（价值观）：用户认同什么？→ 最高级品牌力，用户买的是身份认同

**代表案例**：iPhone — 功能属性（A17芯片/4800万像素）→ 功能利益（流畅不卡/拍照好看）→ 情感利益（高端人群的归属感/科技前沿的优越感）→ 价值观（Think Different/设计至上/简约美学）

**怎么用**：问自己四个问题——
1. 我的产品有什么功能？
2. 这些功能给用户带来什么好处？
3. 用户使用时感受到什么？
4. 我的品牌代表什么价值观？用户会为此骄傲吗？

**在 Skill 中的应用**：
- 模块19（核心价值梳理）：三层价值金字塔（功能价值→情感价值→社会价值）即品牌金字塔的实战版
- 模块14-17（品牌价值体系）：使命/愿景/价值观/人格化是金字塔顶层的具体落地
- 模块8（四种定位方法）：情绪型定位对应第三层，意义型定位对应第四层

## 4. 蓝海战略 — 别在红海里卷

**一句话总结**：不是和竞争对手抢蛋糕，是做一个新蛋糕。通过消除、减少、增加、创造四个动作，重新定义行业边界。

**核心逻辑**：
- 红海 = 竞争激烈、利润微薄、同质化严重
- 蓝海 = 开创没有竞争的新市场空间
- 核心不是技术创新，是价值创新——同时追求差异化和低成本
- 四步动作框架：消除（哪些行业默认规则没必要？）→ 减少（哪些可以降到行业标准以下？）→ 增加（哪些可以大大超过行业标准？）→ 创造（哪些行业从未提供？）

**代表案例**：黄尾葡萄酒 — 消除了（产地/年份/口感的复杂度要求）、减少了（SKU数量，只有红白两款）、增加了（易饮口感/有趣包装）、创造了（"入门级易饮葡萄酒"新品类）→ 美国市场增长最快的葡萄酒品牌

**怎么用**：问自己四个问题——
1. 这个行业有哪些"约定俗成"其实用户不在乎？
2. 哪些地方可以做得比竞品少但用户不介意？
3. 哪些关键维度要远远超过竞品？
4. 有什么是竞品从来没做过但用户会喜欢的？

**在 Skill 中的应用**：
- 模块8（四种定位方法）：切割｜品类占位型定位 = 蓝海战略的定位实现
- 模块18（产品定义）：MVP范围界定中"不做"的清单，就是蓝海的"消除"和"减少"
- 模块19（核心价值梳理）：差异化价值方向的选择，本质是蓝海的"增加"和"创造"

## 5. 4P营销理论 — 营销的四个支柱

**一句话总结**：产品（Product）、价格（Price）、渠道（Place）、推广（Promotion），四个东西相互关联，营销就是不断在这四个维度上找最优解。

**核心逻辑**：
- Product：解决什么问题？功能/设计/体验
- Price：定什么价？传递什么信号？（高价=高端信号，低价=渗透策略）
- Place：在哪卖？怎么触达用户？
- Promotion：怎么让人知道？广告/内容/口碑/活动
- 四个P不是独立的——产品定了价格才能定，价格定了渠道才能选，渠道定了推广才能做

**代表案例**：完美日记 — Product（平价高品质彩妆）、Price（百元以下，低门槛）、Place（线上种草+线下体验店全渠道）、Promotion（小红书+抖音+直播，避开传统广告）

**怎么用**：做每个营销决策时，检查四个P是否对齐——
- 高端产品 + 低价渠道 = 不匹配
- 年轻化产品 + 传统渠道 = 不匹配
- 高价值产品 + 纯促销推广 = 伤害品牌

**在 Skill 中的应用**：
- 4P是整个战略执行篇（模块18-26）的总览框架
- Product → 模块18/19/20（产品策略）
- Price → 模块21（定价策略）
- Place → 模块26（渠道策略，在传播规划中涉及）
- Promotion → 模块22/23/24/25（传播建设策略）

## 6. AISAS模型 — 用户决策全链路

**一句话总结**：用户从知道你到购买分五步——注意（Attention）→ 兴趣（Interest）→ 搜索（Search）→ 行动（Action）→ 分享（Share）。每一阶段需要不同的内容和触点。

**核心逻辑**：
- 传统AIDA模型已过时——互联网时代的用户会主动搜索、看评价、买后分享
- 每个阶段用户行为不同，品牌任务不同，关键指标不同
- S（搜索）和 S（分享）是互联网时代新增的关键环节——搜索是被说服前的最后一步，分享是让用户成为传播者

**五阶段速查**：

| 阶段 | 用户行为 | 品牌任务 | 内容类型 |
|------|---------|---------|---------|
| Attention | 被动接收信息 | 抓住眼球 | 广告/短视频/热点 |
| Interest | 主动了解 | 激发好奇 | 种草内容/教程 |
| Search | 主动搜索 | 提供答案 | SEO内容/评测 |
| Action | 下单购买 | 消除顾虑 | 促销/信任背书 |
| Share | 分享传播 | 创造话题 | 惊喜体验/社交货币 |

**代表案例**：完美日记 — 注意（小红书+抖音大规模种草）→ 兴趣（试色对比+化妆教程）→ 搜索（关键词布局，搜索"平价眼影"必出现）→ 行动（直播间逼单+低门槛首单）→ 分享（产品设计适合拍照发朋友圈）

**怎么用**：确保在每个AISAS阶段，都有对应的内容和触点覆盖。尤其不要忽略S（搜索）和S（分享）——这两个是互联网时代的胜负手。

**在 Skill 中的应用**：
- 模块5（消费者旅程地图）：AISAS是旅程地图的另一种表达，互补使用
- 模块23（内容营销）：转化逻辑链（场景种草→痛点共振→方案呈现→验证闭环）与AISAS同构
- 模块24（社群建设）：Share阶段是社群裂变的底层逻辑

## 7. MVP最小可行性产品 — 完成比完美重要

**一句话总结**：别追求一次性完美，先做一个最小版本（功能少但体验不打折），扔出去验证市场假设，再快速迭代。

**核心逻辑**：
- 做产品前有很多假设——用户有这个需求吗？愿意付费吗？喜欢这种交互吗？
- MVP = 用最小成本验证最大假设
- MVP不是粗制滥造——核心功能必须做好，非核心功能可以砍掉
- MVP不是终点——上线后收集反馈、快速迭代

**代表案例**：Dropbox — 没先开发产品，先做了一个3分钟演示视频，发布后几万人注册等待测试 → 验证了"用户真的需要云同步" → 然后才开始开发真正产品

**常见误区**：
- MVP ≠ 粗制滥造（功能少但体验不差）
- MVP ≠ 半成品（是完整的精简版，能独立解决核心问题）
- MVP ≠ 做完了就不管了（是迭代的起点）

**怎么用**：
1. 找到最不确定的核心假设
2. 设计最简单的验证方式（不一定要写代码——可以是一个视频、一个落地页、一场预售）
3. 收集反馈，快速调整方向

**在 Skill 中的应用**：
- 模块18（产品定义）：MVP范围界定（必须做/可以做/不做）的直接理论来源
- 模块26（年度规划）：Q1 "产品MVP打磨" 的底层逻辑
- 冷启动策略选型矩阵：MVP思维贯穿所有5条冷启动路径

## 8. PMF产品市场匹配 — 产品好不如匹配对

**一句话总结**：产品再好，市场不需要也是白搭。找到PMF后，增长会自己发生——用户主动来、留得住、愿意付费、还帮你推荐。

**核心逻辑**：
- 没找到PMF = 拼命推也推不动
- 找到PMF = 用户自己找上门
- PMF可测量：留存率>40%、NPS>50、自然增长>50%、付费转化率>5%

**PMF四阶段**：
| 阶段 | 特征 | 应该做什么 |
|------|------|-----------|
| 产品前PMF | 只有想法 | 做MVP验证需求 |
| 产品后PMF | 有产品，市场反应一般 | 找早期用户，快速迭代 |
| 渠道PMF | 产品好但不知怎么获客 | 测试渠道，找可规模化的方式 |
| 增长PMF | 产品渠道都找到了 | 加大投入，快速扩张 |

**代表案例**：YouTube — 最初是视频约会网站，用户不买账 → 发现用户更爱上传分享各种视频 → 转型为通用视频平台 → PMF出现 → 10亿美元被Google收购

**怎么用**：
1. 找 must-have 需求（用户没有这个会很难受）
2. 找愿意付费的人群
3. 持续迭代直到找到匹配

**在 Skill 中的应用**：
- 模块18（产品定义）：产品定义的第一步就是判断PMF阶段
- 模块18 MVP界定：MVP的目标就是验证PMF
- Phase 3A 能力校验：判断用户当前处于PMF哪个阶段，决定策略重点
- 模块26：Q1-Q2的核心任务本质是"从产品前PMF到产品后PMF"

```
黄金圈法则（Why）
    ↓ 指导
STP（为谁做、做什么定位）
    ↓ 指导

    ↓ 落实到

    ↓ 通过
MVP → PMF（先验证、再匹配、再放大）
    ↓ 设计用户旅程
AISAS（注意→兴趣→搜索→行动→分享）
    ↓ 竞争视角

```
